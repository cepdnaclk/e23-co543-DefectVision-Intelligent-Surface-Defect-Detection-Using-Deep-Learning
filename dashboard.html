Method comparison

PatchCore (WideResNet50 memory bank) vs. the from-scratch autoencoder, on the three held-out categories. PatchCore leads on every metric and every category — the gap is largest on carpet, a texture class where reconstruction error stops being a useful anomaly signal.

Backbone ablation: ResNet18 vs. WideResNet50

PatchCore's memory-bank backbone traded off: WideResNet50 edges out ResNet18 on localization quality (PRO) by ≤1.3 points, but costs 17–34% more inference time. ResNet18 is the better call under a latency/memory budget.

Explainability quality

Grad-CAM (autoencoder) and occlusion sensitivity (PatchCore) attributions scored against the ground-truth defect mask over 20 anomalous test images per category. Neither method localizes as tightly as detection AUROC would suggest — both operate at a coarser spatial resolution than the raw anomaly map.

Known limitations

Failure modes worth flagging before deployment — see the README for full detail.

Autoencoder fails on texture

Carpet image AUROC (0.319) is worse than random guessing — reconstruction error doesn't separate defects from normal texture variation when "normal" is already irregular-looking.

PatchCore border artifacts

PatchCore's anomaly map sometimes peaks at image borders strongly enough to compete with the true defect region — a real deployment risk, not just an artifact of the explanation method.

Coarse explanations

Both attribution methods run at a coarser spatial resolution (16×16) than the anomaly map itself, so top-5% IoU stays modest even where detection AUROC is excellent.
